# Top-level category browse, no per-tag pages

The MVP exposes category and tag-like filters in the discovery list, derived from fields such as type, reward type, region, and vendor.

It also includes one top-level `/{locale}/categories` browse surface because the accepted UI needs a separate category-oriented view. This page groups opportunities by user goal and field combinations; it does not introduce a CMS taxonomy or a separate data source.

The MVP still does not create one route per category, tag, vendor, or reward type until search demand or SEO evidence justifies those generated pages.
